Important Notes About Supported Platforms

As a result of a significant fall-off in the number of customers using 32-bit Linux platforms, we have decided to drop support for this platform in our next release. We have also chosen not to create MATLAB or R interfaces for our 32-bit Linux port. If you are one of the few customers who currently runs Gurobi on a 32-bit Linux machine, we recommend that you start making arrangements to transition to a different platform.

We should also mention that, as of May of 2011, the Python Software Foundation no longer supports Python 2.5. As a result, we have dropped support for Python 2.5 in this Gurobi release.

Release Notes for Version 5.0

Gurobi 5.0 contains a number of new features and capabilities. Users of previous versions will typically not need to make any changes to their programs to use the new version. The new capabilities are described here.

  • Quadratic constraint support: You can now add Second-Order Cone constraints, rotated Second-Order Cone constraints, and general Quadratic Constraints to an optimization model. Continuous models containing these constraints are solved using a new barrier SOCP solver. Mixed-integer models are solved using a branch-and-cut algorithm, employing either QCP node relaxations or an outer-approximation algorithm. Gurobi 5.0 includes new routines for adding, querying, and removing quadratic constraints, as well as a number of new attributes and parameters for managing such constraints. The new attributes related to quadratic constraints are:
    • NumQConstrs: The number of quadratic constraints in the model.
    • NumQCNZs: The number of non-zero coefficients in the quadratic constraints.
    • IsQCP: Indicates whether the model has quadratic constraints.
    • QCRHS: The right-hand side value for a particular quadratic constraint.
    • QCSense: The sense for a particular quadratic constraint.
    • QCName: The name of a particular quadratic constraint.
    • QCPi: The dual multiplier for a particular quadratic constraint.
    • QCSlack: The constraint slack for a particular quadratic constraint.
    • IISQConstr: Indicates whether a particular quadratic constraint participates in the computed Irreducible Inconsistent Subsystem (IIS).
    The new parameters are:
    • BarQCPConvTol: Convergence tolerance for the barrier algorithm when solving a QCP.
    • PreQLinearize: Controls linearization of Q matrices in the quadratic constraints or a quadratic objective.
    • QCPDual: Determines whether dual variable values are computed for QCP models.
    • MIQCPMethod: Determines whether outer approximation is used to solve an MIQCP model.
  • Lazy constraints: You can now add constraints to a MIP model in a lazy fashion. In other words, rather than having to specify all constraints up front, you can now add additional constraints during the branch-and-cut search (through a callback routine). Previous Gurobi versions allowed you to add cutting planes during the branch-and-cut search, but these new constraints were not allowed to cut off feasible solutions. Set the new DualReductions parameter to zero when you wish to add lazy constraints.
  • Feasibility relaxation: Gurobi now provides a routine that will create a feasibility relaxation model for an infeasible model. This related model will find a solution that minimizes the violation of the original constraints.
  • MATLAB® interface: You can now use Gurobi to optimize a model built within the MATLAB environment.
  • R interface: You can now use Gurobi to optimize a model built within the R statistical computing environment.
  • Simplex warm-starting from a solution: You can now warm-start the simplex method using a primal and dual solution vector. In previous versions, warm-starting was only possible if you had a simplex basis. Use the new PStart and DStart attributes to provide the start solution.
  • Retrieve a variable or constraint by name: You can now quickly retrieve a constraint or variable using its name.
  • Barrier homogeneous algorithm: We now provide a homogeneous algorithm in the barrier solver. This version should be used when you are likely to ask the barrier solver to solve infeasible or unbounded models. Use the new BarHomogeneous parameter to select this.
  • New concurrent optimizer control: You can now control the algorithms used by the different threads in the concurrent optimizer when solving LP and QP models. Use the new GetConcurrentEnv and DiscardConcurrentEnvs routines (the names will vary slightly, depending on the language you are using) to set up different environments for different threads.
  • New KappaExact attribute: We now provide both exact and approximate condition number estimates for the optimal simplex basis. Use Kappa if you want a quick estimate, or KappaExact if you want to spend the sometimes substantial time required to compute the exact condition number.
  • Additional logging control: The new LogToConsole parameter allows you to shut off console logging. You can now choose to send the Gurobi log to a file, to the console, to both, or to neither.
  • Additional information about token licenses: Users of Gurobi token licenses can now use the gurobi_cl --tokens command to obtain information about how many tokens are currently in use, and who is using them.

Code Conversion from Version 4.6

As mentioned earlier, programs that were originally written for version 4.6 of the Gurobi Optimizer will typically require no code changes to use version 5.0. There are a few routines that have changed, though:
  • GetVarNodeRel/GetVarSolution/SetVarSolution: These C++/Java/.NET callback routines are now handled as overloaded signatures of GetNodeRel/GetSolution/SetSolution, respectively.
